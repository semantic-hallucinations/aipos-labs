в консоли создаем контейнер

это уже работает, но для пбз тебе надо еще подключить твой  sql документ. Для хранения данных в докер контейнерах существуют  volume(тома). В докер композе прописано его создание, тебе остается только его заполнить.
при условии что контейнер работает и твои данные из .env и docker compose такие же как у меня запускаем вот эту команду.(это для проведения соединения)
docker exec -it postgres_container psql -U postgres_user -d postgres_db
если тебя кинет в какую то левую консоль после команды введи в нее \q
докер запускаем с настройкой в имя_контейнера юзер имя_юзера детечт мод(чтобы контейнер работал не занимая консоль) название_бдшки
потом эту
docker cp тут_полный_путь_до_файла.sql postgres_container:/tmp/имя_файла.sql
это создает тот самый  volume
пробуем выполнить файл чтобы создать таблички и продуры
docker exec -it postgres_container psql -U postgres_user -d postgres_db -f /tmp/pbz.sql
